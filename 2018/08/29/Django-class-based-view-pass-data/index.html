问题简介

出于自学Django框架的目的，在刷完了Django for Beginners后我开始尝试搭建个人博客。因为沿用了Django for Beginners一书的格式，我用Django.views中的view class来创建subclass，并在urls.py中用as_view来生成相应的页面。但是当想在view class中进行筛选操作时遇到了问题。

比如当我想实现传入一种文章类别来筛选出属于该类别的文章时，我需要传入该类别的key，并通过与传入值比较是否匹配筛选出属于该类别的文章后将其数据传出，随后利用模版对其进行渲染，对于function-based的实现来说，类别的id作为参数传入是非常直观的，但是对于class-based的实现，获取id的地方并不是那么的一目了然。
