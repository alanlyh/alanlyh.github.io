JavaScript Async Function Return Type

Posted on 2018-10-03 | In JavaScript |

问题简介

最近在写microservice的课程project时尝试使用了JavaScript的“async/await”语法来实现一些asynchronous function，使用下来的观感就是写得真的爽，不得不说这样的写法相较于使用promise更容易且代码可读性更高（虽然用promise的写法已经很易读了）。

但是同时在写async function时，我遇到了两种不同的情况，一种是function直接return一个promise，然后在调用这一function时用await得到结果；而另一种情况则是在function前加async，在function中通过await从一系列asynchronous操作得到结果，最后return这一结果。具体例子如下：

在第一种情况下，因为返回的是一个promise，我们可以直接通过await得到结果；但是在第二种情况下我们看到的返回值并不是一个promise，但仍然能使用await来得到执行的结果。对此我产生了一点疑问。

问题解决

我在stack overflow上找到的一个问题和回答很好地解决了我的疑问

async/await implicitly returns promise?

Nathan Wall的回答中提到asynchronous function总是返回promise，如果我们没有明确指定返回promise，返回值将被自行包装成promise，例如上述情况2中，我们看到的return value是result，但实际return的是Promise.resolve(result)。
